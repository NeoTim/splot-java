Class Overview

Interface for a Functional Endpoint.

A Functional Endpoint (FE) can be thought of as an object that represents a set of functionally related properties and methods. A physical device hosts one or more FEs.

Examples of the relationship between physical devices and FEs:

  • A smart light bulb might host a single FE that can be used to control the state of the light bulb.
  • A smart power strip might host one FE for every power outlet on the strip.
  • A proprietary wireless sensor gateway might host one (or more) FEs for every wireless sensor.

All class methods that act on individual properties are type-safe because the value type is baked into the PropertyKey: using the wrong class will cause a compiler error.

Implementation Guidance

If you need to implement a functional endpoint, you have two options:

  • Subclass LocalFunctionalEndpoint (of one of its subclasses). This is useful if you need to control something local. It is straightforward to use and can give you a lot of additional functionality (like scenes and transitions) for free. However, it is (usually) not appropriate to use these abstract classes to implement non-local functionality, such as for a wireless sensor — for that consider the next option.
  • Implement the FunctionalEndpoint interface directly. This might be reasonable if your functional endpoint is very simple, or if you want to expose functionality provided off device such as a wireless sensor or a Zigbee light. This option gives you the most implementation flexibility but is significantly more work to implement and test than the previous option: so if you are implementing a functional endpoint which controls local behaviors, consider the previous option instead.

Once an instance of a functional endpoint is available, you can then host that functional endpoint using a Technology, allowing other devices to use it.

Example Usage

Fetching a property value

To fetch the value of a property on a FunctionalEndpoint instance, you use the fetchProperty(PropertyKey, Modifier...) method. The most simple case would look like this: 

Boolean onOffValue = fe.fetchProperty(OnOffTrait.STAT_VALUE).get();

In this case we are immediately calling get() in order to make the call synchronous and immediately return a value. However, we can make this call asynchronous by simply adding a listener to the returned future: 


 Executor executor = ...;
 ListenableFuture<Boolean> future = fe.fetchProperty(OnOffTrait.STAT_VALUE);
 future.addListener(() -> {
     Boolean onOffValue = future.get();

     /* do stuff with onOffValue here */
 }, executor);

Setting a property value

Setting (and incrementing, toggling, etc.) a property value follows the same pattern as fetching a property value: the method returns a ListenableFuture. However, in these cases the ListenableFuture doesn't wrap around a value: its purpose is simple to provide a way to synchronize on the completion of the operation or to cancel the operation. For example, to synchronously turn fe on we could do the following: 

fe.setProperty(OnOffTrait.STAT_VALUE, true).get();

Alternatively, if we wanted this property change to transition over two seconds, we would add a modifier: 

fe.setProperty(OnOffTrait.STAT_VALUE, true, Modifier.duration(2)).get();

If we don't care about synchronizing to the value being committed, we could even omit the call to get(): 

fe.setProperty(OnOffTrait.STAT_VALUE, true);

However, in this case we would have no way of knowing if the operation was successful or not. This is why if you want to change a value asynchronously you would normally add a listener to the future to handle any exceptions: 


 Executor executor = ...;

 ListenableFuture<?> future = fe.setProperty(OnOffTrait.STAT_VALUE, true);

 future.addListener(() -> {
      try {
          future.get();

      } catch (InterruptedException x) {
          /* See https://goo.gl/ZNZVem */
          Thread.currentThread().interrupt();
          return;

      } catch (java.util.concurrent.CancellationException x) {
          /* Handle cancellation here */
          return;

      } catch (java.util.concurrent.ExecutionException x) {
          /* Handle the exception here */
          return;
      }

      /* Success, do stuff here */
 }, executor);

Getting notifications about individual property value changes

You an use registerPropertyListener(Executor, PropertyKey, PropertyListener) to register a PropertyListener instance to be invoked whenever a property changes value. 


 Executor executor = ...;

 PropertyListener<Boolean> listener = (fe, key, value) -> {
      System.out.println(key + " on " + fe + " just changed to " + value);
 };

 fe.registerPropertyListener(executor, OnOffTrait.STAT_VALUE, listener);

 /* do stuff here */

 fe.unregisterPropertyListener(OnOffTrait.STAT_VALUE, listener);

public abstract ListenableFuture<?> applyProperties (Map<String, Object> properties)

Asynchronously change the value of multiple properties at once.

To add properties to a Map in a type-safe way, use the putInMap(Map, T) method.

The caller can optionally block until completion by calling the get() method, or can be notified of completion by adding a listener via addListener(Runnable, Executor).

The applied properties must all be from the same section. The behavior of passing properties from different sections is undefined.

The returned future may throw one of the following checked exceptions (as the cause to a java.util.concurrent.ExecutionException) when completed:

  • PropertyException, or one of its subclasses:
    • PropertyWriteOnlyException
    • PropertyNotFoundException
    • PropertyOperationUnsupportedException
    • InvalidPropertyValueException
  • TechnologyException if there was a technology-specific problem
However, it this functional endpoint is a group, then this method's future will generally not throw PropertyException for state section properties.

Parameters
properties a map keyed by the property name containing the new values for those properties.
Returns
  • a future that can be used to block execution until the action is complete, cancel the action, or monitor the action asynchronously for completion

public abstract Map<String, Object> copyCachedSection (Section section)

Synchronously return a Map containing the cached values of all of the properties in the given section on this functional endpoint.

Note that if this is a "local" Functional Endpoint (isLocal() returns true ), then the returned values are the actual values of the properties instead of cached values.

The key for the returned Map is the name of the property as returned by getName(). To fetch individual properties from the map in a type-safe way, use coerceFromMap(Map).

Returns
  • a Map containing the cached values of all of the properties in the given section

public abstract ListenableFuture<Boolean> delete ()

Deletes this Functional Endpoint, if possible. Note that not all functional endpoints can be deleted. If the delete operation cannot be performed, the future will indicate an exception. The returned future may throw one of the following checked exceptions (as the cause to a java.util.concurrent.ExecutionException) when completed:

Returns
  • a future object returning a boolean. If the boolean is true, the object was successfully deleted. If the boolean is false, the object is not deletable.

public abstract ListenableFuture<T> fetchProperty (PropertyKey<T> key, Modifier... modifiers)

Asynchronously fetch the value of a specific property.

The fetched value of the property is obtained by calling get() on the returned ListenableFuture, which will block execution until the value is obtained or an exception is thrown. The caller can be notified when the value has been successfully obtained (and, thus, the call to get() is then guaranteed not to block) by adding a listener via addListener(Runnable, Executor).

The returned future may throw one of the following checked exceptions (as the cause to a java.util.concurrent.ExecutionException) when completed:

  • PropertyException, or one of its subclasses:
    • PropertyWriteOnlyException if the property is write-only
    • PropertyNotFoundException if the property isn't present on this fe
  • TechnologyException if there was a technology-specific problem

public abstract T getCachedProperty (PropertyKey<T> key)

Synchronously return the most recent cached value of a specific property.

Note that if this is a "local" Functional Endpoint (isLocal() returns true ), then the returned values are the actual values of the properties instead of cached values.

Note that this method offers no way to differentiate the following cases where null is returned:

  • The property hasn't been cached yet.
  • The property isn't supported by this functional endpoint.
  • The value of the property is actually null.
If such ambiguity is unacceptable, you will need to either use the fetchProperty(PropertyKey, Modifier...) method or inspect the maps returned from copyCachedSection(Section).

Parameters
key the key for the property to fetch
Returns
  • the cached value of the property, or null if the property is not cached.
