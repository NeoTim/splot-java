Group

implements FunctionalEndpoint
com.google.iot.m2m.base.Group

Class Overview

Interface for managing a group. A group is a collection of FunctionalEndpoints which can be controlled as a single FunctionalEndpoint.

The state section of a group functional endpoint is special in that any operation performed on properties in that section will be applied to those properties on all of the group's members.

By comparison, the configuration and metadata sections belong to the group object itself. Thus, it is not possible to change a configuration or metadata property across the members of a group in a single operation like you can with state properties.

The behavior of reading state properties on a group is currently undefined by the splot object model and thus the behavior is technology-specific. This may change at some point.

All groups must implement at least the following traits:

  • BaseTrait, for name, uid, etc.
  • GroupTrait, to provide an in-band interface for adding/removing members.

Groups are referenced by their UID, which for groups is called a GroupId (or GID). Unlike UIDs on other FunctionalEndpoints, the GroupID is immutable and thus cannot be changed after group creation. Use the META_NAME property to associate a human-readable name with a group.

public abstract ListenableFuture<Void> addMember (FunctionalEndpoint fe)

Adds a member to the Group.

The added member must be associated (either hosted or native) with this group's technology, otherwise the returned future will fail with UnacceptableFunctionalEndpointException. Some technologies cannot add arbitrary FunctionalEndpoints to a group. In those cases, this operation will also fail with UnacceptableFunctionalEndpointException.

The returned future will throw GroupNotAvailableException if this group has been deleted or is otherwise no longer available.

Adding groups to other groups that are native to the same Technology instance is prohibited. Doing so will immediately throw IllegalArgumentException. Adding groups to groups hosted on different technologies is not prohibited per-se but may be restricted by a particular technology.

Parameters
fe the functional endpoint to add to this Group
Returns
  • a ListenableFuture used to cancel, track completion, or retrieve an exception thrown while processing. The exact status of the membership addition will be undefined if canceled before complete.
Throws
IllegalArgumentException if fe is a Group that is native to this Group's Technology

public abstract ListenableFuture<Set<FunctionalEndpoint>> fetchMembers ()

Fetches the member functional endpoints of the group. This is not always possible, in those cases (assuming there was no underlying exception at play) the returned future may return 'null' from get().

If isReliable() and either canUseGroup() or canAdministerGroup() return true, then this method is guaranteed to successfully return all of the members of the group.

The returned future will throw GroupNotAvailableException if this group has been successfully deleted (for example, by a call to delete()). if this group has been deleted or is otherwise no longer available.

public abstract boolean isReliable ()

Indicates if this group can reliably apply changes to all members. Non-reliable groups make a best-effort to ensure that all members have received any changes, but dropped packets can lead to some devices not receiving the changes.

When a group is reliable, it may multicast the change initially but MUST soon follow up with each member individually to ensure that the change was received.

Returns
  • true if this group can be considered reliable, false otherwise.
