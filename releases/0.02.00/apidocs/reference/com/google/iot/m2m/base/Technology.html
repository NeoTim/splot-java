Class Overview

Interface for discovering, interacting with, and hosting FunctionalEndpoints.

With the exception of LocalTechnology, implementations of this interface allow you to interact with devices that implement the associated technology using the Splot API.

For example, a hypothetical Zigbee-based technology would allow you to discover and use Zigbee-based devices as FunctionalEndpoints, whereas an SMCP-based technology would allow you to use the Splot API to discover and use SMCP-based devices.

Some technologies also support hosting functional endpoints (such as subclasses of LocalFunctionalEndpoint or even functional endpoints that are obtained from other unrelated technologies) to other devices. This can be used to expose local functionality to other devices on the technology, or even to implement cross-iot-technology bridges.

In addition to the tasks described above, instances of this interface are the primary mechanism for managing *groups*: All instances of Group have an associated Technology instance. Not all technologies support groups, but you can create local groups to implement group behavior at the expense of efficiency using LocalTechnology, which supports forming groups of arbitrary functional endpoints.

While not required, all technologies should implement PersistentStateInterface to allow them to store persistent state.

Relationships with Functional Endpoints

The relationship between any given functional endpoint and a given technology can be described by one of the following terms: native, hosted, or unrelated:

  • A native functional endpoint is owned and managed entirely by a Technology instance, and often represents functionality which is implemented on a different device. You can determine this programmatically using isNative(FunctionalEndpoint).
  • A hosted functional endpoint is not owned or managed by the Technology. Instead, the technology "hosts" the functionality of the functional endpoint for other devices to use. You can determine this programmatically using isHosted(FunctionalEndpoint).
  • An unrelated functional endpoint has no relationship with the technology. Such a functional endpoint cannot be used by the technology or participate in groups hosted by the technology. You can determine this programmatically using isAssociatedWith(FunctionalEndpoint), which will return false if unrelated. The method host(FunctionalEndpoint) can be used to make an unrelated functional endpoint hosted.

In most cases, a functional endpoint is described by only one of the above three labels. However, Group objects (a special interface that extends FunctionalEndpoint) are always native to a Technology, but are are often also hosted by the same Technology, too! This is because groups logically exist on more than one device, so they can be both native and (if any hosted functional endpoints are members) hosted.

public abstract ListenableFuture<Group> createNewGroup ()

Creates a new empty Group with a randomly-generated group identifier. The returned group is NOT automatically hosted, which is required to add hosted functional endpoints to the group. If you are only adding native functional endpoints, then there is no need to host the group.

Note that some technologies may have additional methods for specifying more details about group creation. For example, some technologies support the creation of groups that have exclusive membership. This method will return a group that best matches a "normal" group that has a few arbitrary limitations as possible.

Returns
  • A ListenableFuture for retrieving the newly created group

public abstract ListenableFuture<Group> fetchOrCreateGroupWithId (String groupId)

Looks up a Group object based on a groupId. If one could not be found, a new one is created with the given groupId. Note that, much like createNewGroup(), if a new group is created it is not automatically hosted.

groupId the group identifier of the group to fetch
Returns
  • a future containing a Group with the given group identifier or null if this Technology does not support group creation with arbitrary groupId strings

public abstract URI getNativeUriForFunctionalEndpoint (FunctionalEndpoint fe)

Returns the native URI of the given functional endpoint. The functional endpoint MUST be associated with this technology.

The returned URI must be either an absolute URL (include schema and possibly authority) or a absolute-path-only URL (No schema or authority). The format and scheme of the URI is native (technology specific).

Note that the returned URI is relative to this device/technology. If you intend to set this URI as a property value on a non-local FunctionalEndpoint, you must first change the URI to be relative to that FunctionalEndpoint using getRelativeUriForFunctionalEndpoint(FunctionalEndpoint, URI).

public abstract URI getRelativeUriForFunctionalEndpoint (FunctionalEndpoint fe, URI uri)

Converts the given URI that is relative to this Technology to be relative to the given FunctionalEndpoint. The returned URI may then be set as a property value on the given FunctionalEndpoint.

Default Implementation Details

The default implementation of this method will return the URI unchanged in either of the following two cases:

  • URI is absolute, including scheme component
  • URI is a local absolute path AND isHosted(FunctionalEndpoint) isHosted(fe)} returns true

In the second case:

  • If the URI is not a local absolute path with no scheme, UnknownResourceException is thrown.
  • If isHosted(fe) returns false, then UnassociatedResourceException is thrown.

Parameters
fe The FunctionalEndpoint to convert the URI to be relative to.
uri The URI to convert.
Returns
  • a URI that is relative to the given FunctionalEndpoint.
Throws
UnassociatedResourceException if the given FunctionalEndpoint is not associated with this Technology.
UnknownResourceException if the given URI can't be converted.

public abstract boolean isHosted (FunctionalEndpoint fe)

Determines if the functional endpoint is hosted by this technology or not.

The default implementation uses copyHostedFunctionalEndpointSet() to make this determination, and properly handles indicating child functional endpoints as hosted if their parent (as returned by getParentFunctionalEndpoint()) is hosted (up to four levels deep).

If you override this method, you should use the following code as a starting point: 

FunctionalEndpoint fe;
 FunctionalEndpoint parent;
 int parentLimit = 4;

 do {
     if (IS_FE_HOSTED(fe)) {
         return true;
     

     parent = fe.getParentFunctionalEndpoint();

     if (parent == null) {
         break;
     } else {
         fe = parent;
     }
 } while (parentLimit-- != 0);

 return IS_FE_HOSTED(fe);
 }
Where IS_FE_HOSTED(fe) is a method that performs a simple non-recursive hosted check.

Parameters
fe The functional endpoint to check
Returns
  • true if the functional endpoint is currently hosted by this technology, false otherwise.

public abstract void unhost (FunctionalEndpoint fe)

Unhosts the given functional endpoint if it was previously hosted.

If the given functional endpoint was not hosted by this technology, nothing happens. Calling this method will remove the functional endpoint from any groups associated with this technology. Any saved state about this functional endpoint that was stored by this technology will be forgotten.
