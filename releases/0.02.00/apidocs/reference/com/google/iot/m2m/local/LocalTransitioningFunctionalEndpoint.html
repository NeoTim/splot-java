Class Overview

Abstract class for more easily implementing local FunctionalEndpoints that support smooth transitions between property values, as well as scenes.

By subclassing this class instead of LocalSceneFunctionalEndpoint, transition support will be largely implemented for you automatically. Like LocalFunctionalEndpoint, Subclasses of this class should NOT be used to implement non-local FunctionalEndpoints: in those cases a class directly implementing the methods of FunctionalEndpoint should be used.

If transitions don't make sense for your functional endpoint (for example, if it is an input rather than an output), then you should subclass LocalFunctionalEndpoint (or LocalFunctionalEndpoint if you still need scenes) instead.

public synchronized Map<String, Object> copyPersistentState ()

Retrieves a copy of the state for the instance that should be saved to nonvolatile storage. When reconstructing the associated instance, call initWithPersistentState(Map) to restore this state.

The data contained in the returned Map should be considered opaque unless specified otherwise by the implementing technology.

The types for the values contained in the map are only guaranteed to be faithfully restoreable if they are one of the following types:

  • Integer
  • Double
  • Float
  • byte[]
  • String
  • Map<String,Object>, where Object is also one of these acceptable types
  • java.util.List<Object>, where Object is also one of these acceptable types
This allows the object to be easily serialized to CBOR.

This method MUST be thread safe.

Returns
  • the persistent state of this instance

public ListenableFuture<Map<String, Object>> fetchSection (Section section, Modifier... mods)

Asynchronously fetch a Map containing the values of all of the properties in the given section on this functional endpoint.

The key for the returned Map is the name of the property as returned by getName(). To fetch individual properties from the map in a type-safe way, use coerceFromMap(Map).

The returned future may throw one of the following checked exceptions (as the cause to a java.util.concurrent.ExecutionException) when completed:

  • TechnologyException if there was a technology-specific problem.

Parameters
mods The modifiers to apply to this operation, like transitionTarget().
Returns
  • a future to retrieve a map containing all reported property values in a given section (Can also be cancelled or monitored asynchronously for completion)
