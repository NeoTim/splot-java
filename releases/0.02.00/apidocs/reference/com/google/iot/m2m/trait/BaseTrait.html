Class Overview

The base trait is the one trait which is required on all Functional Endpoints. It contains information about the model, manufacturer, and identifier, as well as administratively configurable properties like the administrative name, administrative id, and the hidden flag.

Fields

public static final PropertyKey<Boolean> META_HIDDEN

Hidden flag.

This property is a simple boolean flag indicating if this FE should be hidden from administrative views.

public static final PropertyKey<String> META_MANUFACTURER

Manufacturer name.

Unique to the manufacturer. This property identifies the specific model of the device hosting this FE. Note that this field is not for the marketing name: use ‘prod’ for that.

public static final PropertyKey<String> META_MODEL

The model identifier for this functional endpoint, unique to the manufacturer.

It identifies the specific model of the device hosting this FE. Note that this field is not for the marketing name: use ‘prod’ for that.

public static final PropertyKey<String> META_NAME

Administrative, human-readable name of the functional endpoint.

After a factory reset, this is set to a descriptive default value by the manufacturer. This value does not need to be unique.

public static final PropertyKey<Boolean> META_PERMANENT

Permanent flag.

This property indicates if this FE is a permanent fixture on this device or if it can be administratively created or deleted.

public static final PropertyKey<Map<String, String>> META_PRODUCT_NAME

The localized names of the product.

This property is a dictionary containing the localized names of the product in at least one language. The key to the dictionary is the locale code (like "en" or "jp"), and the value is the localized name for that locale.

public static final PropertyKey<String> META_SERIAL

Manufacturer unique-identifier/serial-number.

This is typically the FE index appended to the serial number of the device. The presence of this field is optional and may be omitted for privacy purposes.

public static final PropertyKey<String> META_SW_VERSION

Software version description string.

This is often the version of the software running on the device, but in the case of a bridge may differ.

public static final PropertyKey<String[]> META_TRAIT_PROFILES

Supported trait profiles.

This property identifies which trait profiles this functional endpoint implements. Trait profiles define what the minimum implementation requirements are for specific types of functionality, such as lights. The first listed profile is intended to best describe the functionality of the FE, with subordinate profiles listed subsequently.

public static final PropertyKey<String> META_TRAIT_URI

Property key for the URI that identifies the specification used to implement this trait.

This property is present on all traits.

public static final PropertyKey<String> META_UID

Administrative unique identifier of the FE.

Identifies the function of the FE. Rules may reference this identifier instead of a direct path in order to make replacement easier. After a factory reset, the value of this field is set to a random UID. This value must be unique.

public static final PropertyKey<String> STAT_TRAP

Current trap condition.

When not null, contains a string indicating the current error/trap condition. This property will automatically revert to null when the error/trap condition has been cleared, which depends on the type of device. This would be used to indicate things like power overload, door obstructed, manual override, or battery too low.
