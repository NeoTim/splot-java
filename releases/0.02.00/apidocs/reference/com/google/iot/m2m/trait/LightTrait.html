Trait for describing and controlling various characteristics of a variety of illuminaries.

This trait is typically paired with OnOffTrait and LevelTrait.

public static final PropertyKey<float[][]> META_PRIMARIES

An array describing the primaries used on the light (up to six) in the CIE xyY colorspace. Each primary is described by a three-element array that contains the x, y, and Y values respectively for the primary. The Y component is normalized to where the maximum brightness of the light is Y=1.0.

public static final PropertyKey<float[]> STAT_CHROMA_XY

The ‘x’ and ‘y’ CIE chromaticity coordinates for the current color.

This property is REQUIRED on full-color lights and OPTIONAL on color-temperature lights.

If the values written to this property are out of gamut, then the state when read will reflect an in-gamut approximation.

Writing to this property changes STAT_MODE to MODE_CIE_XY.

public static final PropertyKey<Float> STAT_CIE_CS

'C*' from CIELCh colorspace, analogous to saturation. Normalized between 0.0 and 1.0. Writing to this property changes ‘mode’ to ‘LCh’. Out-of-gamut values are not reflected: When read or mutated, this value is NOT adjusted to be in-gamut.

Writing to this property changes STAT_MODE to MODE_CIE_LCH.

public static final PropertyKey<Float> STAT_CIE_HD

'h°' from CIELCh colorspace, analogous to hue. Normalized between 0.0 and 1.0. Writing to this property changes ‘mode’ to ‘LCh’. The value is circular: Incrementing past 1.0 automatically loops back around to 0.0. Decrementing below 0.0 automatically loops back around to 1.0. When read or mutated, this value is NOT adjusted to be in-gamut.

Writing to this property changes STAT_MODE to MODE_CIE_LCH.

public static final PropertyKey<Float> STAT_CIE_LS

A normalized, perceptually uniform value representing lightness. Defined as L* from the CIELAB/CIELCh colorspaces, except normalized between 0 and 1. A value of 0.0 represents zero light output, a value of 1.0 represents full light output, and all values in between represent perceptually uniform levels of lightness. This definition necessitates that this property's functionality encapsulate that of both STAT_VALUE and STAT_VALUE.

This property is different from that of STAT_VALUE because a value of 0.0 in STAT_VALUE represents the lowest-possible light output, whereas a value of 0.0 in this property represents zero light output. Both attempt to be somewhat perceptually uniform.

There is often a significant discontinuity between the lowest possible light output level and the light's "off" state. As a special case for all lightness levels below META_MIN_DIM, the transition from the lowest possible light output to “off” is calculated using META_MIN_DIM and is at the perceptual half-way point between no light output and the value for this property associated with a STAT_VALUE of 0.0.

Writing to this property has no effect on STAT_MODE.

public static final PropertyKey<String> STAT_EFFECT

Identifies the current special effect in progress.

The interpretation of this field is vendor-specific, with the following exceptions: null indicates no effect is in use, EFFECT_COLORCYCLE indicates that the hue of the light is cycling through the colors, and EFFECT_CANDLE indicates that the light is emulating the flicker of a candle.

Vendors MAY implement other effects with unique string identifiers. If this property is implemented, META_EFFECTS MUST also be implemented.

public static final PropertyKey<Float> STAT_MIREDS

The current color temperature (absolute when written, correlated when read) in Mireds.

Implementing this property is REQUIRED on color-temperature lights and RECOMMENDED on full-color lights.

When this property is written to, the following changes occur:

  • The light is configured to emit "white" light at the given color temperature.
  • STAT_MODE (if implemented) changes to MODE_COLOR_TEMP
  • STAT_WHITEPOINT (if implemented) changes to this property's value
  • STAT_CIE_CS becomes zero.

If STAT_MODE is set to MODE_COLOR_TEMP, reading this property will yield the current color temperature of the white light that is being emitted.

If STAT_MODE is NOT set to MODE_COLOR_TEMP, reading this property should yield either the correlated color temperature of the light that is being emitted or null.

About Mireds

Mireds are a perceptually-uniform way to indicate color temperature. This is an important property to ensure that transitions appear to be smooth and natural.

You can easily convert from Kelvin to Mireds using the following formula:

M = 1000000 / K

Because of the reciprocal relationship between Mireds and Kelvin, the exact same formula also works for converting Mireds to Kelvin.

public static final PropertyKey<String> STAT_MODE

Enumeration indicating the last color system used.

Can be one of the following values: MODE_COLOR_TEMP, MODE_SRGB, MODE_CIE_XY, MODE_CIE_LCH.

This property is REQUIRED if the other implemented properties have side effects that would set this property to more than one specific value.

Internally, this property is used to determine which properties need to be stored in order to be able to properly recall the state later on.

public static final PropertyKey<float[]> STAT_SRGB

The sRGB red, green, and blue components, normalized to values between 0.0 and 1.0 (inclusive). Uses the real composite sRGB gamma curve. Setting this value will change the color of the light to match the contained values.

Implementing this property is RECOMMENDED on full-color lights.

If STAT_WHITEPOINT is supported, use that as the reference whitepoint, otherwise use D65.

The read value is NOT cropped to fit into the gamut of the device, but individual values MAY be limited to the range of 0.0-1.0 or reflect the loss of precision from conversion to the internal representations. If the gamut of the device is larger than the sRGB gamut, then values outside of the range of 0.0-1.0 MAY be allowed.

The value read SHOULD read as null if STAT_MODE is not MODE_SRGB, otherwise it reports the last value written.

Setting a value of all zeros MUST cause STAT_VALUE to become false. Setting a value with any non-zero component MUST cause the STAT_VALUE to become true.

Writing to this property changes STAT_MODE to MODE_SRGB.

public static final PropertyKey<float[]> STAT_WHITEPOINT

This property controls the whiteppoint used by the STAT_SRGB and STAT_CIE_HD/STAT_CIE_CS properties, identified by CIE xy coordinates.

Implementing this property is RECOMMENDED on full-color lights.
